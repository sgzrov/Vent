Run a quality-aware load test against an already-deployed voice agent. Fires increasing tiers of simultaneous real conversations (10% → 25% → 50% → 100% of target_concurrency) to find exactly WHERE the agent breaks under load. Every concurrent call is a full multi-turn conversation: CallerLLM → TTS → send → collect via VAD → streaming STT → repeat for max_turns (default 6). After all tiers, ALL transcripts are evaluated against eval questions with JudgeLLM. Results include per-tier metrics, severity grading (excellent/good/acceptable/critical), quality degradation curves, and 2-of-3 breaking point detection (error rate + P95 latency + quality drop). Returns a run_id immediately — use voiceci_get_run_status with test_type='load_test' to get results via long-polling. Only works with already-deployed agents (SIP, WebRTC, or websocket with agent_url).
