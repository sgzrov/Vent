Run a load/stress test against an already-deployed voice agent. Sends N concurrent calls with a traffic pattern (ramp, spike, sustained, soak). Measures TTFB percentiles, error rates, and auto-detects breaking point. Returns a run_id immediately — then use voiceci_get_run_status to get results via long-polling (same as voiceci_run_tests). Only works with already-deployed agents (SIP, WebRTC, or websocket with agent_url).
