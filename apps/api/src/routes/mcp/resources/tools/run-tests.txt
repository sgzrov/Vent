Run infrastructure probes and conversation tests against a voice agent using a layered test architecture.

**Layered model:**
- Layer 1 (Infrastructure): 3 quick probes (audio_quality, latency, echo) that return raw metrics — no pass/fail. Configure via `infrastructure` key.
- Layers 2-4 (Conversation): CallerLLM-driven conversations with optional `audio_actions` (interrupt, silence, inject_noise, split_sentence, noise_on_caller) injected at specific turns. Configure via `conversation_tests` key.

Read `voiceci-config.json` from the project root and pass its contents as the `config` parameter. For already-deployed agents (SIP, WebRTC, platform adapters, or agent_url), this queues immediately. For local WebSocket agents with start_command, a relay command is returned.

After getting the run_id, spawn subagents to monitor results via voiceci_get_run_status.
