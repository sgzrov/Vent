Run conversation tests against a voice agent.

Each conversation test includes integrated signal quality analysis (SNR, clipping, energy, drift) — no separate infrastructure probes needed.

Tests use CallerLLM-driven conversations with optional audio_actions (interrupt, silence, inject_noise, split_sentence, noise_on_caller) at specific turns.

Generate a test config inline based on what you need to test, and pass it as the `config` parameter. For deployed agents (SIP, WebRTC, platform adapters, or agent_url), the run queues immediately. For local WebSocket agents with start_command, a relay command is returned — run it in the background.

After getting the run_id, spawn subagents to monitor results via voiceci_get_run_status. Call voiceci_guide_reference for config format details.
